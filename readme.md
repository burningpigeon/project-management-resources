
## Issue Templates

These are templates used to create GitHub issues for keeping track of work. These can be filled out to give the developer(s) a better idea on what they are working on

| Issue Type        | Purpose                                                  |
| ----------------- | -------------------------------------------------------- |
| **Bug**           | Something that is broken or behaves incorrectly.         |
| **Feature**       | A completely new capability or major functionality.      |
| **Enhancement**   | A small improvement or extension to an existing feature. |
| **Refactor**      | Internal code improvements with no user-visible changes. |
| **Documentation** | Documentation additions, fixes, or improvements.         |
| **Test**          | Add or improve automated tests.                          |
